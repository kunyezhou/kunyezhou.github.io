博客搭建过程及教程（顺便功能测试）

发表于 2016-11-25   |   分类于 2016年11月   |  

前言

第一篇博客主要是为了测试和熟悉各种功能，顺便吐槽一下ywt同学的非人类教程，并给真正的小小白白白白白白白白白白提供一份傻瓜教程。写博客大概就是想记录一些东西给自己看，强迫自己归纳和整理，很多内容都存在局限性，希望大家多提建议，多喷一下，让小野早日进步。

首先我们要了解接下来我们的博客是在github pages上套一个Hexo实现的。

PS：ywt同学的教程是在你已经熟悉了linux与git的前提下进行操作的，像我这种工科狗第一次接触，蒙逼了半天才搭好这个平台。

准备工作

  • github账号
    github作为全球最大的基佬交友平台，当然是免费注册的。自己跑去官网注册一个就好了。一定要给自己想好username，因为你的username和博客地址有关。
  • git安装
    git安装当然也是去官网下载了。不需要翻墙，git for Windows
  • node.js安装
    Node.js去官网下载是需要翻墙的。Node.js

Git的配置

配置SSH Key

看看是否存在SSH密钥(keys)

在桌面点击右键出现git bash here，会出现
将代码cd ~/. ssh输入后

检查你本机用户home目录下是否存在.ssh目录

如果，不存在此目录，则进行第二步操作，否则，你本机已经存在ssh公钥和私钥，可以略过第二步，直接进入第三步操作。

创建一对新的SSH密钥(keys)

$ssh-keygen -t rsa -C "your_email@example.com"
#这将按照你提供的邮箱地址，创建一对密钥
Generating public/private rsa key pair.
Enter file in which to save the key (/c/Users/you/.ssh/id_rsa): [Press enter]

直接回车，则将密钥按默认文件进行存储。此时也可以输入特定的文件名，比如/c/Users/you/.ssh/github_rsa

接着，根据提示，你需要输入密码和确认密码（说到这里，如果你很放心，其实可以不用密码，就是到输密码的地方，都直接回车，所以每次push就只管回车就行了。所谓的最安全的密码，就是没有密码 哈哈）。相关提示如下：

Enter passphrase (empty for no passphrase): [Type a passphrase]
Enter same passphrase again: [Type passphrase again]

输入完成之后，屏幕会显示如下信息：

Your identification has been saved in /c/Users/you/.ssh/id_rsa.
Your public key has been saved in /c/Users/you/.ssh/id_rsa.pub.
The key fingerprint is:
01:0f:f4:3b:ca:85:d6:17:a1:7d:f0:68:9d:f0:a2:db your_email@example.com

在GitHub账户中添加你的公钥

运行如下命令，将公钥的内容复制到系统粘贴板(clipboard)中。

clip < ~/.ssh/id_rsa.pub

紧接着要登陆你的github,在网站上将SSH Key添加到账户。在账户的settings下面有SSH Key的字样，Title一栏随便填写，将黏贴板的内容全部复制到Key上即可。

测试

可以输入以下命令，测试设置是否成功。

ssh -T git@github.com

若显示下面的提示，则表示设置成功！

The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

放心的输入yes,就会看到

Hi xxxx! You’ve successfully authenticated, but GitHub does not provide shell access.

设置用户信息

现在你已经可以通过SSH链接到GitHub了，还有一些个人信息需要完善的。
Git会根据用户的名字和邮箱来记录提交。GitHub也是用这些信息来做权限的处理，输入下面的代码进行个人信息的设置，把名称和邮箱替换成你自己的，名字必须是你的真名，而不是GitHub的昵称。

输入以下命令

$ git config --global user.name "cnfeat"//用户名
$ git config --global user.email "cnfeat@gmail.com"//填写自己的邮箱

然后并不会有什么设置成功的提示，没有提示就说明你成功了~

Hexo配置

Hexo安装

输入以下命令进行Hexo的安装，可能没个人的网速不同，如果很慢就等等吧

npm install hexo-cli -g npm install hexo --save

Hexo初始化配置

安装好Hexo之后，挑一个你喜欢的位置，新建一个文件夹，文件夹目录最好不要有中文。点击鼠标右键 git bash here，并输入一下命令

hexo init
npm install

不知道大家网速如何，我是等了他半天，校园网连外网网速不好。如果很慢就等等吧~

Hexo的插件安装

某韬的教程中仅安装了一个小插件就可以搞定博客的配置，实际上也是这样的。但是我在看其他教程中，很多都提到了要安装一坨插件，作为小白白白白白白白白的我，顺手把其他的插件也都安装了，可能会方便日后的使用吧。现在将这两者插件命令陈列如下。

ywt同学的插件（必须安装用来上传的插件）

npm install hexo-deployer-git --save

其他攻略上所建议安装的插件

npm install hexo-generator-index --save
npm install hexo-generator-archive --save
npm install hexo-generator-category --save
npm install hexo-generator-tag --save
npm install hexo-server --save
npm install hexo-deployer-git --save
npm install hexo-deployer-heroku --save
npm install hexo-deployer-rsync --save
npm install hexo-deployer-openshift --save
npm install hexo-renderer-marked@0.2 --save
npm install hexo-renderer-stylus@0.2 --save
npm install hexo-generator-feed@1 --save
npm install hexo-generator-sitemap@1 --save

到了这里，博客的本地部署就完成了。如果顺利的话下面执行如下命令就能在本地查看你的主页了。

hexo g hexo s

将博客部署到Github Pages上

那么现在本地的博客已经搭建起来了，但是我们只可以通过本地连接查看我们的博客。那么我们现在需要做的就是把本地的博客发布到服务器上，让别人也可以连接我们的博客，而Github Pages就帮我完成了这件事情。但是Github Pages的代码就是寄存在Github上面的。那么接下来我们需要在Github上面创建一个新的项目。

创建一个新的库

登陆github，然后点击New repository。建立一个新的库，库的名字要写你的

用户名.github.io

举一个栗子，我注册的时候用户名写的是kunyezhou,那么我的Github Pages对应的库的名字就只能是kunyezhou.github.io。

将Hexo文件更新到库

进入刚才所创建的库，随后点击clone。可以选择SSH或者HTTP。建议选择SSH，复制到剪贴板，大概的内容如下：

git@github.com:kunyezhou/kunyezhou.github.io.git

修改站点配置文件

在刚才下载Hexo的文件夹下，找到_config.yml的文件，打开并修改其最后几行。
将

# Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
type:

修改为

# Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
type:git
repo： //刚才在剪贴板上的SSH地址
branch： master

注意冒号是英文格式的（看我对大家多贴心）。随后再执行下列命令即可完成Hexo与Github Pages的链接。

Hexo g -d

在这个过程中会有一些警告提醒你，不用理会。可能我的网速很慢，等待的过程比较漫长，大概等了六七分钟才好。大家不要着急，总会好的：)

#Hexo #闲扯
